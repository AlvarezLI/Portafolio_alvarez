Proyecto 1: Modelo de Edibilidad de Hongos

Este proyecto es parte de mi proyecto final para un curso de ciencia de datos, donde clasifiqué diferentes hongos según su edibilidad.

Fuente del Conjunto de Datos

El conjunto de datos utilizado es el Mushroom Classification dataset de Kaggle, que proporciona una descripción completa de las características de los hongos.

Técnicas de Preprocesamiento de Datos

  • Comprobaciones de Valores Nulos: Se garantizó la integridad del conjunto de datos al manejar adecuadamente los valores faltantes.
  • Codificación de Variables Categóricas: Se transformaron las características categóricas en valores numéricos para un entrenamiento efectivo del machine learning.

Algoritmos de Machine Learning

  • Random Forest Classifier
  • Support Vector Machine (SVM)
  • Logistic Regression

Proyecto 2: Modelo de Predicción del Clima

En este proyecto, realicé un análisis profundo de un conjunto de datos sintético diseñado para clasificar diferentes tipos de clima.

Técnicas de Preprocesamiento de Datos

  • Manejo e Imputación de Valores Nulos: Se abordaron los valores faltantes para mantener la calidad del conjunto de datos.
  • Codificación de Variables Categóricas: Se convirtió las variables categóricas en un formato adecuado para el entrenamiento del modelo.
  • Eliminación de Outliers: Se aplicó el método del rango intercuartílico (IQR) para eliminar los outliers.

Algoritmos de Machine Learning

  • Random Forest Classifier
  • Gradient Boosting Machine (GBM)
  • Logistic Regression
  • Decision Tree Classifier

Optimización y Evaluación del Modelo

  • Accuracy: Se midió la corrección general de las predicciones.
  • Confusion Matrix: Se evaluó el rendimiento de los modelos de clasificación.

Se compararon los modelos para identificar el mejor ajuste y se realizó un ajuste de hiperparámetros para mejorar los resultados.

Proyecto 3: Análisis de Obras Públicas en Excel

En este proyecto, realicé un extenso análisis de datos de obras públicas en Argentina, con el objetivo de obtener información sobre la distribución y estado de las inversiones en infraestructura a nivel regional.

Fuente del Conjunto de Datos

El conjunto de datos fue obtenido del Mapa de Inversiones del Ministerio de Obras Públicas de Argentina, que proporciona información detallada sobre las obras públicas en varias provincias y su estado de avance.

Técnicas de Análisis y Preprocesamiento de Datos

  • Limpieza y Validación de Datos: Se eliminaron errores y se validaron tanto valores cualitativos como cuantitativos para garantizar la precisión del análisis.
  • Tablas Dinámicas: Se utilizaron para facilitar la exploración de datos y obtener respuestas a preguntas clave como el número de obras por tipo, estado y ubicación geográfica.
  • Segmentación de Datos: Se añadieron segmentadores para permitir la segmentación dinámica de los datos por categorías como tipo de obra, año, estado de ejecución y provincia.

Visualización y Tablero

Se creó un tablero visual interactivo en Excel con gráficos y segmentadores que permiten una rápida visualización de la distribución y estado de las obras públicas:

  • Distribución de Obras por Provincia: Resalta las provincias con mayor inversión en obras.
  • Clasificación de Obras por Estado: Muestra el avance de las obras (planificadas, en progreso, completadas).
  • Análisis por Tipo de Obra: Identifica las categorías prioritarias de obras en inversión pública.

Resultados Clave

Este análisis proporcionó una visión general del estado de las obras públicas en Argentina, destacando las regiones con mayor y menor inversión, así como los tipos de proyectos más comunes y su nivel de avance. Estas ideas son valiosas para comprender las tendencias de inversión y las posibles áreas de mejora en la gestión de obras públicas.
